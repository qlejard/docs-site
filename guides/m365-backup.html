Architecture de Microsoft 365 Backup, RPO/RTO, différence rétention native vs backup indépendant — 🔴 Critique

Contexte — Pourquoi une sauvegarde Microsoft 365 ?

💡
Analogie : La rétention native de Microsoft 365, c'est la poubelle de bureau — vous retrouvez ce que vous avez jeté récemment, mais si le bureau brûle, tout est perdu. Microsoft 365 Backup, c'est une assurance incendie : une copie indépendante, dans un coffre séparé, qui permet de tout reconstruire à un instant donné.

Microsoft 365 applique le modèle de responsabilité partagée (Shared Responsibility Model) : Microsoft garantit la disponibilité de l'infrastructure (réplication, datacenters), mais le client reste responsable de ses données. Sans sauvegarde dédiée, votre organisation n'est protégée que par les mécanismes de rétention native — insuffisants face à une suppression malveillante, un ransomware chiffrant les fichiers, ou une purge administrative accidentelle.

Analogie : Propriétaire vs locataire — Microsoft est le propriétaire de l'immeuble (structure, ascenseurs, parties communes). Vous êtes le locataire : vos meubles, vos documents, vos objets de valeur sont sous votre responsabilité. Si vous perdez vos clés ou si vos affaires sont volées, le propriétaire ne vous remboursera pas.

⚠️ Piège fréquent

⚠️
Piège n°1 : Confondre « rétention » et « backup ». La rétention native de Microsoft 365 (corbeille, litigation hold, retention policies Purview) n'est pas un backup. Elle ne protège pas contre :
  • Une suppression par un admin compromis (qui peut aussi purger les éléments retenus)
  • Un ransomware qui chiffre les fichiers (les versions chiffrées écrasent les précédentes au-delà de la limite de versioning)
  • Un besoin de restauration au-delà de la fenêtre recycle bin (93 jours SPO, 14-30 jours Exchange)
⚠️
Piège n°2 : Penser que Microsoft 365 Backup est automatique. Le service doit être explicitement activé dans le Microsoft 365 admin center, et des backup policies doivent être créées pour chaque service (Exchange, SharePoint, OneDrive). Sans cela, le score reste à 0.

Connect-MgGraph -Scopes "BackupRestore.ReadWrite.All"

# Vérifier si le service M365 Backup est activé

    -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore"

    Write-Host "[OK] Microsoft 365 Backup est ACTIF" -ForegroundColor Green
    Write-Host "  Service ID   : $($backupService.id)"
    Write-Host "  Status       : $($backupService.status)"

# Lister les protection policies existantes
$policies = Invoke-MgGraphRequest -Method GET `
    -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies"

if ($policies.value.Count -eq 0) {
    Write-Host "[WARN] Aucune policy de backup configurée" -ForegroundColor Yellow
} else {
    $policies.value | ForEach-Object {
        Write-Host "[OK] Policy : $($_.displayName) | Type : $($_.@odata.type) | Status : $($_.status)" -ForegroundColor Green

Sauvegarde et restauration des boîtes aux lettres Exchange — emails, calendrier, contacts — 🔴 Critique

Contexte — Pourquoi sauvegarder Exchange Online ?

Exchange Online stocke les emails, calendriers, contacts et tâches — le cœur de la communication d'entreprise. La rétention native (corbeille 14-30 jours, litigation hold) ne couvre pas tous les scénarios de perte. Microsoft 365 Backup pour Exchange offre un point-in-time recovery avec un RPO d'environ 10 minutes, permettant de restaurer une boîte aux lettres complète ou des éléments spécifiques à un instant précis.

💡
Analogie : Sans M365 Backup, la protection Exchange c'est comme un filet de sécurité avec des trous : la corbeille « Deleted Items » (14j), la corbeille de récupération « Recoverable Items » (14-30j), et le litigation hold (illimité mais pas un point-in-time). Le backup ajoute une machine à remonter le temps : restaurez n'importe quel moment des 365 derniers jours.

⚠️ Piège fréquent

⚠️
Piège : Compter sur le Litigation Hold comme backup Exchange. Le litigation hold conserve les éléments supprimés indéfiniment, mais :

PowerShell — Créer une policy backup Exchange via Microsoft Graph 
# ────────────────────────────────────────────────────────────
# S2 — Créer une backup policy Exchange Online
# Prérequis : M365 Backup activé (voir S1)
# ────────────────────────────────────────────────────────────

Connect-MgGraph -Scopes "BackupRestore.ReadWrite.All"

# Créer la policy Exchange (protège toutes les mailboxes par défaut)
$body = @{
    "@odata.type" = "#microsoft.graph.exchangeProtectionPolicy"
    "displayName" = "Exchange Backup Policy - All Mailboxes"
} | ConvertTo-Json

$policy = Invoke-MgGraphRequest -Method POST `
    -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies" `
    -Body $body `
    -ContentType "application/json"

Write-Host "Policy créée : $($policy.displayName) | ID : $($policy.id)"

# Ajouter des mailboxes spécifiques au scope (optionnel — ciblage)
$users = @("user1@contoso.com", "user2@contoso.com", "shared-mailbox@contoso.com")
foreach ($user in $users) {
    $userObj = Get-MgUser -Filter "mail eq '$user'"
    $unitBody = @{
        "@odata.type" = "#microsoft.graph.mailboxProtectionUnit"
        "directoryObjectId" = $userObj.Id
    } | ConvertTo-Json

    Invoke-MgGraphRequest -Method POST `
        -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies/$($policy.id)/mailboxProtectionUnits" `
        -Body $unitBody `
        -ContentType "application/json"
    Write-Host "  Mailbox ajoutée : $user" -ForegroundColor Green
}

# Activer la policy
Invoke-MgGraphRequest -Method POST `
    -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies/$($policy.id)/activate"
Write-Host "[OK] Policy Exchange activée" -ForegroundColor Green

Contexte — Pourquoi sauvegarder SharePoint Online ?

SharePoint Online héberge les sites d'équipe, les bibliothèques de documents, les listes, les pages intranet et les données partagées de l'organisation. La rétention native (corbeille 93 jours, versioning) protège contre les suppressions accidentelles récentes, mais ne couvre pas la restauration complète d'un site à un point précis dans le temps, ni la protection contre une purge administrative ou un ransomware qui écraserait toutes les versions.

💡
Analogie : La corbeille SharePoint (93 jours) c'est comme un filet de sécurité sous un trapéziste — il rattrape les chutes récentes. Microsoft 365 Backup, c'est un parachute de secours : même si le filet est coupé (purge admin), vous pouvez restaurer l'intégralité du site à l'instant T de votre choix.

Gestion des Coûts M365 Backup
Modèle de tarification, estimation par scénario, optimisation des coûts — 🔵 Informatif

Contexte — Modèle de coût pay-per-use

Microsoft 365 Backup facture au stockage consommé : 0,15 $/Go/mois (tarif indicatif, mai 2026). Le volume correspond aux données protégées à la source (taille des mailboxes + sites + drives). Script All-in-One — Configuration & Audit M365 Backup
Script exécutable pour activer et vérifier Microsoft 365 Backup
⚠️
Prérequis : PowerShell 7+, modules Microsoft.Graph et ExchangeOnlineManagement et Microsoft.Online.SharePoint.PowerShell installés. Compte avec rôle Microsoft 365 Backup Administrator + Exchange Administrator + SharePoint Administrator.
PowerShell — Script All-in-One M365 Backup 
# ════════════════════════════════════════════════════════════
#  SCRIPT ALL-IN-ONE — Microsoft 365 Backup — Audit & Config
#  Version 1.1 — Mai 2026
# ════════════════════════════════════════════════════════════

$totalScore = 0
$maxTotal = 100
$results = @()

# ── [1/10] Connexion ──
Write-Host "`n[1/10] Connexion aux services M365..." -ForegroundColor Cyan
Connect-MgGraph -Scopes "BackupRestore.ReadWrite.All","Directory.Read.All"
Connect-ExchangeOnline
Connect-SPOService -Url "https://<tenant>-admin.sharepoint.com"

# ── [2/10] S1 — Architecture (15 pts) ──
Write-Host "`n[2/10] S1 — Vérification service M365 Backup..." -ForegroundColor Cyan
$backup = Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore"
$s1 = 0
if ($backup.status -eq "enabled") { $s1 += 10; Write-Host "  [+10] Service actif" -ForegroundColor Green }
$policies = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies").value
if ($policies.Count -ge 3) { $s1 += 5; Write-Host "  [+5] 3+ policies définies" -ForegroundColor Green }
$results += [pscustomobject]@{Section="S1";Score=$s1;Max=15}
$totalScore += $s1

# ── [3/10] S2 — Exchange (15 pts) ──
Write-Host "`n[3/10] S2 — Exchange Online Backup..." -ForegroundColor Cyan
$s2 = 0
$exoPolicies = $policies | Where-Object { $_.'@odata.type' -like "*exchange*" }
if ($exoPolicies) {
    $s2 += 5; Write-Host "  [+5] Policy Exchange existe" -ForegroundColor Green
    $activeExo = $exoPolicies | Where-Object { $_.status -eq "active" }
    if ($activeExo) { $s2 += 5; Write-Host "  [+5] Policy active" -ForegroundColor Green }
    $units = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies/$($exoPolicies[0].id)/mailboxProtectionUnits").value
    if ($units.Count -gt 0) { $s2 += 5; Write-Host "  [+5] $($units.Count) mailboxes protégées" -ForegroundColor Green }
}
$results += [pscustomobject]@{Section="S2";Score=$s2;Max=15}
$totalScore += $s2

# ── [4/10] S3 — SharePoint (12 pts) ──
Write-Host "`n[4/10] S3 — SharePoint Online Backup..." -ForegroundColor Cyan
$s3 = 0
$spoPolicies = $policies | Where-Object { $_.'@odata.type' -like "*sharePoint*" }
if ($spoPolicies) {
    $s3 += 4; Write-Host "  [+4] Policy SharePoint existe" -ForegroundColor Green
    $activeSpo = $spoPolicies | Where-Object { $_.status -eq "active" }
    if ($activeSpo) { $s3 += 4; Write-Host "  [+4] Policy active" -ForegroundColor Green }
    $units = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies/$($spoPolicies[0].id)/siteProtectionUnits").value
    if ($units.Count -gt 0) { $s3 += 4; Write-Host "  [+4] $($units.Count) sites protégés" -ForegroundColor Green }
}
$results += [pscustomobject]@{Section="S3";Score=$s3;Max=12}
$totalScore += $s3

# ── [5/10] S4 — OneDrive (10 pts) ──
Write-Host "`n[5/10] S4 — OneDrive for Business Backup..." -ForegroundColor Cyan
$s4 = 0
$odbPolicies = $policies | Where-Object { $_.'@odata.type' -like "*oneDrive*" }
if ($odbPolicies) {
    $s4 += 4; Write-Host "  [+4] Policy OneDrive existe" -ForegroundColor Green
    $activeOdb = $odbPolicies | Where-Object { $_.status -eq "active" }
    if ($activeOdb) { $s4 += 3; Write-Host "  [+3] Policy active" -ForegroundColor Green }
    $units = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/protectionPolicies/$($odbPolicies[0].id)/driveProtectionUnits").value
    if ($units.Count -gt 0) { $s4 += 3; Write-Host "  [+3] $($units.Count) drives protégés" -ForegroundColor Green }
}
$results += [pscustomobject]@{Section="S4";Score=$s4;Max=10}
$totalScore += $s4

# ── [6/10] S5 — Policies (10 pts) ──
Write-Host "`n[6/10] S5 — Policies centralisées..." -ForegroundColor Cyan
$s5 = 0
$activePolicies = $policies | Where-Object { $_.status -eq "active" }
$types = $activePolicies | ForEach-Object { $_.'@odata.type' }
if ($types -like "*exchange*") { $s5 += 3 }
if ($types -like "*sharePoint*") { $s5 += 3 }
if ($types -like "*oneDrive*") { $s5 += 2 }
if ($s5 -eq 8) { $s5 += 2; Write-Host "  [+2] Bonus 3/3 services" -ForegroundColor Green }
Write-Host "  S5 score : $s5 / 10"
$results += [pscustomobject]@{Section="S5";Score=$s5;Max=10}
$totalScore += $s5

# ── [7/10] S6 — Restore (10 pts) ──
Write-Host "`n[7/10] S6 — Restore operations..." -ForegroundColor Cyan
$s6 = 0
$exoR = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/exchangeRestoreSessions").value
$spoR = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/sharePointRestoreSessions").value
$odbR = (Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/solutions/backupRestore/oneDriveForBusinessRestoreSessions").value
$allR = @($exoR) + @($spoR) + @($odbR)
if ($allR.Count -gt 0) { $s6 += 4 }
if ($exoR.Count -gt 0 -and $spoR.Count -gt 0 -and $odbR.Count -gt 0) { $s6 += 3 }
$recent = $allR | Where-Object { [datetime]$_.createdDateTime -gt (Get-Date).AddDays(-90) }
if ($recent.Count -gt 0) { $s6 += 3 }
Write-Host "  S6 score : $s6 / 10"
$results += [pscustomobject]@{Section="S6";Score=$s6;Max=10}
$totalScore += $s6

# ── [8/10] S7 — Rétention (8 pts) ──
Write-Host "`n[8/10] S7 — Rétention native vs Backup..." -ForegroundColor Cyan
$s7 = 0
if ($backup.status -eq "enabled") { $s7 += 3 }
if ($s5 -ge 8) { $s7 += 3 }
Write-Host "  S7 score : $s7 / 8 (+2 si matrice documentée)"
$results += [pscustomobject]@{Section="S7";Score=$s7;Max=8}
$totalScore += $s7

# ── [9/10] S8 — Sécurité (8 pts) ──
Write-Host "`n[9/10] S8 — Sécurité & Compliance..." -ForegroundColor Cyan
$s8 = 0
try {
    $auditCfg = Get-AdminAuditLogConfig
    if ($auditCfg.UnifiedAuditLogIngestionEnabled) { $s8 += 3 }
} catch { Write-Host "  [Info] UAL check requires EXO connection" }
$ba = Get-MgDirectoryRoleMember -DirectoryRoleId (Get-MgDirectoryRole -Filter "displayName eq 'Microsoft 365 Backup Administrator'").Id -ErrorAction SilentlyContinue
if ($ba -and $ba.Count -le 3) { $s8 += 3 }
Write-Host "  S8 score : $s8 / 8 (+2 si MFA/CA)"
$results += [pscustomobject]@{Section="S8";Score=$s8;Max=8}
$totalScore += $s8

# ── [10/10] S9-S10 — Tiers & Coûts (12 pts) ──
Write-Host "`n[10/10] S9-S10 — Vérifications manuelles..." -ForegroundColor Cyan
$results += [pscustomobject]@{Section="S9";Score=0;Max=6}
$results += [pscustomobject]@{Section="S10";Score=0;Max=6}
Write-Host "  S9 : 0/6 (vérif. manuelle — choix backup documenté ?)"
Write-Host "  S10 : 0/6 (vérif. manuelle — estimation coût validée ?)"

# ── Résultat final ──
Write-Host "`n════════════════════════════════════════" -ForegroundColor Cyan
Write-Host "  SCORE TOTAL : $totalScore / $maxTotal" -ForegroundColor Cyan
Write-Host "════════════════════════════════════════" -ForegroundColor Cyan
$results | Format-Table -AutoSize

if ($totalScore -le 15) { $level = "🔴 Aucun backup" }
elseif ($totalScore -le 45) { $level = "🟠 Backup partiel" }
elseif ($totalScore -le 70) { $level = "🟡 Backup fonctionnel" }
elseif ($totalScore -le 90) { $level = "🟢 Backup complet" }
else { $level = "🔵 Exemplaire" }
Write-Host "  Niveau : $level" -ForegroundColor White
