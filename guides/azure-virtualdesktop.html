S1
Architecture & Composants AVD M365 E3 Azure Pay
Host Pools, Application Groups, Workspaces, Reverse Connect, RDP Shortpath — architecture de référence Hub-Spoke + AVD
🚨
Scénario d'attaque : Un attaquant exploite un port RDP ouvert (3389) directement sur une VM de session host accessible depuis Internet. Sans Reverse Connect, sans NSG stricte, la VM est compromise en quelques secondes via bruteforce ou une vulnérabilité RDP (BlueKeep, DejaBlue). Analogie : AVD = "bureau portable sécurisé dans le cloud" — tu te connectes depuis n'importe où mais le bureau reste chez Microsoft. Reverse Connect = "appel sortant sécurisé" — c'est la VM qui appelle ton client, jamais l'inverse. Zéro port entrant ouvert.

💡
Pourquoi chaque paramètre compte :
Host Pool Pooled : sans Pooled multi-session, chaque utilisateur utilise une VM dédiée — coût x4. Windows 10/11 Enterprise Multi-Session est exclusif à AVD et permet 4-10 sessions simultanées (source : Microsoft Learn, "Session host sizing guidance").
Reverse Connect : en mode AVD natif, zéro port entrant. La connexion est établie sortante depuis l'hôte (Microsoft Cloud Security Benchmark NS-1 — AVD non couvert nominativement par CIS Azure Foundations v6.0.0) depuis la VM vers le broker Microsoft via HTTPS 443. Toute architecture ouvrant RDP 3389 entrant contourne cette protection (CIS Microsoft Azure Foundations Benchmark v2.0, section 9.3).
RDP Shortpath UDP : réduit la latence de 20-50ms sur des connexions à haute latence — améliore l'expérience utilisateur pour les applications graphiques. Sans UDP, on reste sur TCP relay via broker Microsoft.
Métadonnées West Europe / France Central : les métadonnées AVD (noms utilisateurs, noms sessions) stockées en East US posent un problème RGPD Art. 44 (transfert hors UE). France Central = résidence données UE.
⚠️
Piège fréquent : activer RDP Shortpath sans ouvrir le port UDP 3478 sortant (STUN/TURN) dans le NSG des session hosts vers le service tag WindowsVirtualDesktop. Résultat : fallback silencieux sur TCP 443 sans avertissement. Vérifier via : Get-NetConnectionProfile + Event Viewer > Microsoft-Windows-RemoteDesktopServices-RdpCoreTS/Operational.

S1.3 — Configuration : création Host Pool Pooled

Azure CLI — Création Host Pool + Application Group + Workspace
# Variables
RG="rg-avd-prod"
LOCATION="westeurope"
HOSTPOOL="hp-avd-prod-pooled"
APPGROUP="ag-avd-prod-desktop"
WORKSPACE="ws-avd-prod"

# Créer le Resource Group
az group create --name $RG --location $LOCATION

# Créer le Host Pool Pooled (Breadth-First load balancing)
az desktopvirtualization hostpool create \
  --name $HOSTPOOL \
  --resource-group $RG \
  --location $LOCATION \
  --host-pool-type "Pooled" \
  --load-balancer-type "BreadthFirst" \
  --max-session-limit 10 \
  --preferred-app-group-type "Desktop" \
  --registration-info expiration-time="$(date -u -d '+24 hours' '+%Y-%m-%dT%H:%M:%S.000Z')" registration-token-operation="Update"

# Créer l'Application Group Desktop
az desktopvirtualization applicationgroup create \
  --name $APPGROUP \
  --resource-group $RG \
  --location $LOCATION \
  --application-group-type "Desktop" \
  --host-pool-arm-path $(az desktopvirtualization hostpool show \
      --name $HOSTPOOL --resource-group $RG --query id -o tsv)

# Créer le Workspace et associer l'Application Group
az desktopvirtualization workspace create \
  --name $WORKSPACE \
  --resource-group $RG \
  --location $LOCATION \
  --application-group-references $(az desktopvirtualization applicationgroup show \
      --name $APPGROUP --resource-group $RG --query id -o tsv)
PowerShell — Création Host Pool via Az.DesktopVirtualization
# Installer le module si absent
Install-Module -Name Az.DesktopVirtualization -Force
Connect-AzAccount

$rg       = "rg-avd-prod"
$location = "westeurope"

# Créer le Host Pool Pooled avec expiration token 24h
$expiry = (Get-Date).AddHours(24).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ss.fffZ")
New-AzWvdHostPool `
    -ResourceGroupName $rg `
    -Name              "hp-avd-prod-pooled" `
    -Location          $location `
    -HostPoolType      "Pooled" `
    -LoadBalancerType  "BreadthFirst" `
    -MaxSessionLimit   10 `
    -RegistrationTokenOperation "Update" `
    -ExpirationTime    $expiry `
    -PreferredAppGroupType "Desktop"

Scénario d'attaque : Credential stuffing sur un compte AVD sans MFA. L'attaquant récupère 10 000 couples login/mot de passe (dark web) et en obtient 50 valides. 💡
Pourquoi cibler les bonnes apps AVD : une politique CA ciblant uniquement « Microsoft Azure Management » ne couvre pas les connexions utilisateurs aux sessions AVD. Il faut cibler Azure Virtual Desktop (9cdead84-a844-4324-93f2-b2e6bb768d07) et, si le SSO sur session host est activé, Windows Cloud Login (270efc09-cd0d-444b-a71f-39af4910ec45). Si l'app s'affiche encore comme Windows Virtual Desktop dans le portail, c'est le même app ID 9cdead84..., pas un deuxième identifiant legacy à ajouter. Ne pas cibler l'app Azure Virtual Desktop Azure Resource Manager Provider (50e95039-b200-4007-bc97-8d5790743a63) pour le MFA utilisateur.

Sign-in frequency : la réauthentification ne coupe pas une session active ; elle s'applique lorsqu'un nouvel access token est requis, par exemple lors d'une nouvelle connexion, d'un refresh de feed ou d'un rebinding de session. L'option Every time n'est documentée que pour Windows Cloud Login quand le SSO est activé.
⚠️
Pièges fréquents :
1. Exclure les comptes de service de MFA sans workload identity protection — utiliser des Managed Identities, pas des user accounts.
2. Oublier Windows Cloud Login quand le SSO AVD est activé : l'utilisateur passe alors la passerelle AVD mais pas la connexion au session host avec le même niveau d'exigence CA.
3. Cibler l'app ARM Provider pour le MFA utilisateur : elle sert au feed et pas à l'authentification utilisateur interactive à la session.
4. PIM sans Access Review : élévation accordée une fois, jamais retirée si la personne change de rôle.

S2.2 — Configuration : Politique CA MFA + Sign-in Frequency

Microsoft Graph PowerShell — CA Policy MFA obligatoire pour AVD
# Prérequis : module Microsoft.Graph
Connect-MgGraph -Scopes "Policy.ReadWrite.ConditionalAccess"

$avdAppId = "9cdead84-a844-4324-93f2-b2e6bb768d07"  # Azure Virtual Desktop
$windowsCloudLoginAppId = "270efc09-cd0d-444b-a71f-39af4910ec45"  # Ajouter si SSO activé sur les session hosts

$includeApps = @($avdAppId)
# Si le SSO AVD est activé, décommentez la ligne suivante :
# $includeApps += $windowsCloudLoginAppId

$caPolicy = @{
    displayName = "[AVD] MFA obligatoire — Tous utilisateurs"
    state       = "enabledForReportingButNotEnforced"  # Commencer en mode audit !
    conditions  = @{
        users        = @{ includeUsers = @("All") }
        applications = @{ includeApplications = $includeApps }
        clientAppTypes = @("all")
    }
    grantControls = @{
        operator        = "OR"
        builtInControls = @("mfa")
    }
    sessionControls = @{
        signInFrequency   = @{ value = 1; type = "hours"; isEnabled = $true }
        persistentBrowser = @{ mode = "never"; isEnabled = $true }
    }
}
New-MgIdentityConditionalAccessPolicy -BodyParameter $caPolicy
Write-Host "Politique CA AVD créée en mode audit — basculer vers 'enabled' après validation"

$avdPolicies = $policies | Where-Object {

💡
Reverse Connect et Shortpath ne sont pas le même chemin : AVD commence toujours par un transport Reverse Connect en TCP 443 vers la passerelle AVD, puis tente un transport UDP si RDP Shortpath est disponible. Si UDP échoue, la session continue automatiquement en TCP. Pour le mode public, Microsoft documente un direct STUN sur ports UDP dynamiques et un relay TURN sur 51.5.0.0/16:3478. Pour le mode managed, il faut activer le listener RDP Shortpath sur les session hosts et autoriser le port entrant 3390/UDP par défaut.

Private Link AVD n'est pas le Private Endpoint Azure Files : pour privatiser le service AVD lui-même, Microsoft utilise des private endpoints distincts global, feed et connection. Si vous activez l'UDP via Private Link, il faut cocher l'opt-in UDP sur le host pool, désactiver les options RDP Shortpath public (STUN/TURN), et, si vous filtrez finement les ports, autoriser la plage TCP dynamique 1-65535 vers le private endpoint connection. Le Private Endpoint Azure Files reste, lui, la bonne pratique pour FSLogix sur SMB 445.
⚠️
Pièges fréquents :
1. Réduire RDP Shortpath à UDP 3478 : ce port concerne TURN ; le direct STUN utilise des ports UDP dynamiques et le mode managed utilise le listener 3390/UDP.
2. Activer Private Link AVD sans opt-in UDP puis espérer Shortpath : sans opt-in explicite, l'UDP via Private Link est bloqué.
3. Laisser STUN/TURN public activés après activation de l'UDP sur Private Link : le portail demande de les désactiver avant sauvegarde.
4. Désactiver le pare-feu Windows sur les session hosts « pour éviter les conflits » avec le NSG. Le NSG filtre au niveau réseau, le Windows Firewall au niveau hôte — ils sont complémentaires.

PowerShell — Script de déploiement AVD Sécurisé (Phases 1/8 à 8/8)
#Requires -Version 7.0
# Script AVD Security Hardening — All-in-One
# Usage : .\avd-hardening.ps1 [-DryRun] [-Phase 1..8]

param(
    [switch]$DryRun = $false,
    [ValidateRange(1,8)][int]$Phase = 0  # 0 = toutes les phases
)

# ===================== [CONFIG] =====================
$RG               = "rg-avd-prod"
$Location         = "westeurope"
$HostPoolName     = "hp-avd-prod-pooled"
$WorkspaceName    = "ws-avd-prod"
$StorageAccount   = "stavdfslogixprod"
$KeyVaultName     = "kv-avd-prod"
$LAWName          = "law-avd-prod"
$VNetName         = "vnet-avd-prod"
$SubnetName       = "snet-avd-sessionhosts"
# ====================================================

function Invoke-Step($description, $scriptBlock) {
    Write-Host "`n[AVD] $description" -ForegroundColor Cyan
    if ($DryRun) { Write-Host "  [DRY-RUN] Étape simulée" -ForegroundColor Yellow; return }
    try { & $scriptBlock; Write-Host "  OK" -ForegroundColor Green }
    catch { Write-Warning "  ERREUR : $_" }
}

# [1/8] Infrastructure — NSG + subnet
if ($Phase -eq 0 -or $Phase -eq 1) {
    Invoke-Step "[1/8] NSG Session Hosts" {
        az network nsg create --name "nsg-avd-sessionhosts" `
            --resource-group $RG --location $Location
        az network nsg rule create --nsg-name "nsg-avd-sessionhosts" `
            --resource-group $RG --name "DenyInternetInbound" `
            --priority 200 --direction Inbound --access Deny `
            --source-address-prefix Internet --destination-port-range "*"
    }
}

# [2/8] MFA + Conditional Access (via Graph PS)
if ($Phase -eq 0 -or $Phase -eq 2) {
    Invoke-Step "[2/8] Conditional Access — MFA AVD" {
        Connect-MgGraph -Scopes "Policy.ReadWrite.ConditionalAccess" -NoWelcome
        # Voir S2 pour le script complet CA policy
        Write-Host "  → Configurer via portail entra.microsoft.com ou script S2"
    }
}

# [3/8] Private Endpoint Azure Files
if ($Phase -eq 0 -or $Phase -eq 3) {
    Invoke-Step "[3/8] Private Endpoint Azure Files FSLogix" {
        az storage account update --name $StorageAccount `
            --resource-group $RG --public-network-access Disabled
        az network private-endpoint create `
            --name "pe-avd-fslogix" --resource-group $RG `
            --vnet-name $VNetName --subnet $SubnetName `
            --private-connection-resource-id $(az storage account show `
                --name $StorageAccount --resource-group $RG --query id -o tsv) `
            --group-id file --connection-name "pe-conn-fslogix"
    }
}

# [4/8] Session Host Hardening (exclusions AV + ASR)
if ($Phase -eq 0 -or $Phase -eq 4) {
    Invoke-Step "[4/8] ASR Rules + Exclusions FSLogix (à exécuter sur chaque session host)" {
        Add-MpPreference -ExclusionExtension "vhd"
        Add-MpPreference -ExclusionExtension "vhdx"
        Add-MpPreference -ExclusionProcess "%ProgramFiles%\FSLogix\Apps\frxccds.exe"
        Add-MpPreference -ExclusionProcess "%ProgramFiles%\FSLogix\Apps\frxsvc.exe"
        Write-Host "  Exclusions AV FSLogix configurées"
    }
}

# [5/8] FSLogix Profile Container
if ($Phase -eq 0 -or $Phase -eq 5) {
    Invoke-Step "[5/8] FSLogix Profile Container" {
        $reg = "HKLM:\SOFTWARE\FSLogix\Profiles"
        If (-Not (Test-Path $reg)) { New-Item $reg -Force | Out-Null }
        Set-ItemProperty $reg Enabled 1 -Type DWord
        Set-ItemProperty $reg DeleteLocalProfileWhenVHDShouldApply 1 -Type DWord
        Set-ItemProperty $reg VHDLocations `
            "\\$StorageAccount.file.core.windows.net\fslogix-profiles" -Type String
    }
}

# [6/8] Diagnostic Settings AVD vers Log Analytics
if ($Phase -eq 0 -or $Phase -eq 6) {
    Invoke-Step "[6/8] Diagnostic Settings AVD" {
        $lawId = az monitor log-analytics workspace show `
            --resource-group $RG --workspace-name $LAWName --query id -o tsv
        $hpId = az desktopvirtualization hostpool show `
            --name $HostPoolName --resource-group $RG --query id -o tsv
        az monitor diagnostic-settings create --name "diag-avd-hostpool" `
            --resource $hpId --workspace $lawId `
            --logs '[{"category":"Connection","enabled":true},{"category":"Error","enabled":true},{"category":"AgentHealthStatus","enabled":true}]'
    }
}

# [7/8] Resource Lock production
if ($Phase -eq 0 -or $Phase -eq 7) {
    Invoke-Step "[7/8] Resource Lock CanNotDelete" {
        az lock create --name "lock-avd-prod" `
            --resource-group $RG --lock-type CanNotDelete
    }
}

# [8/8] Vérification finale
if ($Phase -eq 0 -or $Phase -eq 8) {
    Invoke-Step "[8/8] Rapport de vérification" {
        Write-Host "`n=== VÉRIFICATION AVD HARDENING ===" -ForegroundColor Cyan
        az desktopvirtualization hostpool show --name $HostPoolName `
            --resource-group $RG `
            --query "{Name:name, Type:hostPoolType, ValidationEnv:validationEnvironment}" --output table
        az lock list --resource-group $RG `
            --query "[].{Name:name, Type:level}" --output table
        Get-ItemProperty "HKLM:\SOFTWARE\FSLogix\Profiles" -ErrorAction SilentlyContinue |
            Select-Object Enabled, VHDLocations
    }
}
Write-Host "`n[AVD] Script terminé. Consulter le guide pour les étapes manuelles restantes." -ForegroundColor Green

📖 Glossaire

AcronymeSignificationContexte AVD
AVDAzure Virtual DesktopService Microsoft de virtualisation de postes de travail et d'applications dans Azure. Anciennement "Windows Virtual Desktop" (WVD)
FSLogix—Technologie Microsoft de gestion de profils utilisateurs via conteneurs VHD/VHDX montés dynamiquement au login sur les session hosts AVD
VHD / VHDXVirtual Hard Disk / eXtendedFormat de disque virtuel utilisé par FSLogix pour stocker les profils utilisateurs. VHDX recommandé pour les nouvelles installations
RDP ShortpathRemote Desktop Protocol ShortpathMode de transport AVD utilisant UDP (au lieu de TCP via reverse proxy) pour réduire la latence et améliorer la qualité des sessions interactives
Reverse Connect—Architecture AVD où le session host établit une connexion sortante vers le service AVD (pas d'ouverture de port entrant requis sur les firewalls)
MSIX App Attach—Technologie Microsoft permettant d'attacher des applications packagées au format MSIX dynamiquement aux sessions AVD, sans installation sur l'image OS
WDACWindows Defender Application ControlPolitique de contrôle d'applications Windows (successeur d'AppLocker) qui s'applique à tous les processus, y compris les services SYSTEM
ASRAttack Surface ReductionEnsemble de règles Microsoft Defender for Endpoint (MDE) qui bloquent des comportements malveillants courants (macros Office, LSASS dump, scripts obfusqués)
SIG / ACGShared Image Gallery / Azure Compute GalleryService Azure de stockage et distribution d'images VM versionnées avec réplication géographique. Nouveau nom : Azure Compute Gallery
AIBAzure Image BuilderService Azure permettant de créer automatiquement des images VM personnalisées via des templates JSON (basé sur HashiCorp Packer)
CAConditional Access / Accès ConditionnelPolitiques Microsoft Entra ID qui évaluent les conditions de connexion (utilisateur, appareil, IP, risque) avant d'autoriser l'accès aux ressources
PIMPrivileged Identity ManagementService Entra ID P2 permettant l'activation JIT (Just-In-Time) des rôles privilégiés avec durée limitée et workflow d'approbation
MDEMicrosoft Defender for EndpointSolution EDR Microsoft déployée sur les session hosts AVD pour la protection, détection et réponse aux menaces
NSGNetwork Security GroupPare-feu stateless Azure appliqué sur les subnets ou interfaces réseau des VM pour filtrer le trafic entrant et sortant
CMKCustomer-Managed KeysOption Azure permettant au client de gérer ses propres clés de chiffrement (via Key Vault) au lieu des clés gérées par Microsoft (PMK)
CAEContinuous Access EvaluationMécanisme Entra ID qui révoque les tokens d'accès en temps réel lors d'événements critiques (révocation de session, changement de mot de passe)
GRSGeo-Redundant StorageOption de réplication Azure Storage qui copie automatiquement les données vers une région secondaire (ex: WestEurope → NorthEurope)
