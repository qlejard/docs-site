Mode d'emploi de ce guide
Comment utiliser ce document comme référentiel de sécurité et de gouvernance M365 Copilot
📋
Ce document est un référentiel technique complet couvrant la sécurisation de Microsoft 365 Copilot : architecture, prévention du surpartage, classification des données, contrôles d'accès, monitoring et stratégie de déploiement. Il peut être utilisé tel quel en présentation client ou adapté en ajoutant les données spécifiques d'un tenant.
🚨
Risque principal : M365 Copilot hérite des permissions existantes de l'utilisateur. Si un employé a accès à un fichier confidentiel partagé par erreur avec « Everyone except external users », Copilot peut citer et résumer ce fichier dans ses réponses. Analogie : Copilot est un assistant ultra-documenté — il a accès à tout ce que vous avez accès, y compris les tiroirs que vous ne saviez pas ouverts.

1.0 — Contexte

💡
Analogie : Microsoft 365 Copilot est un assistant personnel ultra-documenté qui a lu tous les documents, e-mails et messages Teams auxquels vous avez accès. Il ne voit que ce que vous voyez — mais c'est précisément le problème : si vos permissions sont trop larges, Copilot hérite de cet excès. Imaginez un stagiaire qui aurait accès à tous les fichiers RH parce que quelqu'un a partagé un dossier avec « Everyone » il y a 3 ans.

1.2 — Pourquoi c'est critique

⚠️
Piège fréquent : Beaucoup d'organisations déploient Copilot sans audit préalable des permissions. Le résultat : un utilisateur demande « résume mes projets en cours » et Copilot cite un document confidentiel RH, un plan de restructuration ou un fichier financier auquel l'utilisateur avait un accès théorique via un lien « Everyone except external users » créé il y a des années.
Copilot ne crée pas de faille — il révèle les failles existantes.

2.0 — Contexte

💡
Analogie : L'oversharing, c'est comme un tiroir ouvert dans un bureau partagé. 2.2 — Pourquoi c'est critique

⚠️
Piège fréquent : Croire que « Restricted SharePoint Search » suffit. RSS est un pansement temporaire limité à 100 sites qui empêche Copilot d'indexer les sites non listés. Mais il ne corrige pas les permissions sous-jacentes. Si un utilisateur a un lien direct vers un fichier d'un site non listé dans RSS, il y a toujours accès.
La vraie solution : nettoyer les permissions + déployer RSS en parallèle comme filet de sécurité.

2.2b — Workflow de remédiation du surpartage

Oversharing Remediation Workflow
🔍 Phase 1 — Découverte
• Activer RSS (protection immédiate)
• Exécuter rapport SAM Data Access Governance
• Script PowerShell : lister sites avec « Everyone » / « Anyone » links
• Identifier les top 20 sites à risque
🧹 Phase 2 — Nettoyage
• Révoquer les liens « Anyone » expirés ou inutiles
• Convertir « Everyone except external » en groupes de sécurité spécifiques
• Désactiver l'héritage sur les dossiers sensibles
• Supprimer les permissions directes obsolètes
✅ Phase 3 — Validation
• Re-exécuter le rapport SAM et comparer
• Test Copilot avec un compte pilote : vérifier qu'il ne cite plus de fichiers sensibles
• Ajouter les sites nettoyés dans la allow-list RSS
🔄 Phase 4 — Gouvernance continue
• SAM Site lifecycle policies (notification à 90 jours d'inactivité)
• Access Reviews trimestrielles pour les sites à risque
• Alertes automatiques sur nouvelles expositions « Everyone »
⚠️
Attention : RSS est limité à 100 sites en allowlist. Pour les tenants avec des centaines de sites SharePoint, RSS seul ne suffit pas — il faut combiner avec un nettoyage des permissions (revoke Everyone, conversion en groupes de sécurité) et un contrôle du partage au niveau tenant. De plus, RSS ne protège pas les fichiers OneDrive — seul le contrôle des permissions individuelles couvre ce périmètre.

2.3b — Application (PowerShell / Graph API)

PowerShell — Audit oversharing SharePoint pour Copilot
# ── Prérequis ──
Install-Module Microsoft.Online.SharePoint.PowerShell -Scope CurrentUser -Force
Install-Module PnP.PowerShell -Scope CurrentUser -Force

# ── Connexion SharePoint Online ──
$tenantUrl = "https://TENANT-admin.sharepoint.com"
Connect-SPOService -Url $tenantUrl

# ── 1. Lister les sites avec partage trop ouvert ──
Get-SPOSite -Limit All | Where-Object {
    $_.SharingCapability -ne "Disabled"
} | Select-Object Title, Url, SharingCapability, `
    @{N='Propriétaire';E={$_.Owner}}, `
    LastContentModifiedDate | Sort-Object SharingCapability -Descending |
    Export-Csv -Path "CopilotOversharing_Sites.csv" -NoTypeInformation

# ── 2. Détecter les sites avec liens « Everyone except external users » ──
$risqueSites = @()
Get-SPOSite -Limit All | ForEach-Object {
    $site = $_
    Try {
        Connect-PnPOnline -Url $site.Url -Interactive
        $links = Get-PnPSharingLink -Scope Organization
        if ($links.Count -gt 0) {
            $risqueSites += [PSCustomObject]@{
                Site      = $site.Title
                Url       = $site.Url
                NbLinks   = $links.Count
                Owner     = $site.Owner
            }
        }
    } Catch { Write-Warning "Accès refusé : $($site.Url)" }
}
$risqueSites | Sort-Object NbLinks -Descending | Format-Table -AutoSize

# ── 3. Restreindre le partage au niveau tenant ──
Set-SPOTenant -SharingCapability ExternalUserSharingOnly `
    -DefaultSharingLinkType Direct `
    -PreventExternalUsersFromResharing $true `
    -RequireAnonymousLinksExpireInDays 14

# ── 4. Désactiver le partage « Anyone » sur un site spécifique ──
Set-SPOSite -Identity "https://TENANT.sharepoint.com/sites/Finance" `
    -SharingCapability ExistingExternalUserSharingOnly

# ── 5. Activer RSS via PowerShell (SharePoint Online Management Shell) ──
Set-SPOTenant -IsRestrictedSharePointSearchEnabled $true
# Ajouter les sites autorisés
Add-SPORestrictedSearchAllowedList -Sites @(
    "https://TENANT.sharepoint.com/sites/Intranet",
    "https://TENANT.sharepoint.com/sites/Policies",
    "https://TENANT.sharepoint.com/sites/IT-Knowledge"
)

# ── 6. Lister les sites orphelins (sans propriétaire) ──
Get-SPOSite -Limit All | Where-Object {
    $_.Owner -eq "YOURORG\spo-system" -or $_.Owner -like "*system*"
} | Select-Object Title, Url, Owner, LastContentModifiedDate

3.0 — Contexte

💡
Analogie : Les sensitivity labels sont des étiquettes de classification sur des dossiers papier. Sans étiquette, impossible de savoir si un document est public ou confidentiel — et Copilot traite tout de la même manière. Avec des labels, vous définissez qui peut accéder, si le chiffrement s'applique, et si Copilot peut utiliser le contenu comme source.
⚠️

Sans labels, Copilot ne fait aucune distinction entre un document public et un plan stratégique confidentiel. De plus, Copilot hérite du label du contenu source le plus restrictif dans ses réponses (label inheritance).

Script All-in-One — Hardening Copilot Complet
Script exécutable pour appliquer toutes les best practices de sécurisation Copilot
⚠️
Attention : Ce script réalise un audit en lecture seule. Il ne modifie aucune configuration. Revoyez les résultats avant d'appliquer des changements. Exécutez-le avec un compte Global Admin ou Security Admin.
PowerShell — Audit Copilot All-in-One
# ══════════════════════════════════════════════════════════
# AUDIT COPILOT ALL-IN-ONE — Lecture seule
# Prérequis : Microsoft.Graph, ExchangeOnlineManagement,
#            PnP.PowerShell (si SharePoint audit souhaité)
# ══════════════════════════════════════════════════════════

param(
    [string]$TenantId,
    [string]$OutputPath = ".\CopilotAudit_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
)

New-Item -Path $OutputPath -ItemType Directory -Force | Out-Null

# ── Connexions ──
Write-Host "[1/8] Connexion Microsoft Graph..." -ForegroundColor Cyan
Connect-MgGraph -Scopes @(
    "User.Read.All","Policy.Read.All","Directory.Read.All",
    "ReportSettings.Read.All","AuditLog.Read.All",
    "InformationProtectionPolicy.Read.All"
)

# ── S1 : Architecture — Licences Copilot ──
Write-Host "[2/8] S1 — Inventaire licences Copilot..." -ForegroundColor Cyan
$copilotSku = Get-MgSubscribedSku | Where-Object { $_.SkuPartNumber -match "Copilot" }
$copilotSku | Select-Object SkuPartNumber, ConsumedUnits,
    @{N='Total';E={$_.PrepaidUnits.Enabled}} |
    Export-Csv "$OutputPath\S1_Licences.csv" -NoTypeInformation

# ── S2 : Oversharing — Sites Everyone/EEEU ──
Write-Host "[3/8] S2 — Analyse oversharing (Graph)..." -ForegroundColor Cyan
$sites = Invoke-MgGraphRequest -Uri "https://graph.microsoft.com/v1.0/sites?`$top=100" -Method GET
$sites.value | Select-Object displayName, webUrl |
    Export-Csv "$OutputPath\S2_Sites.csv" -NoTypeInformation

# ── S3 : Labels — Sensitivity Labels publiés ──
Write-Host "[4/8] S3 — Labels de sensibilité..." -ForegroundColor Cyan
$labels = Invoke-MgGraphRequest -Uri "https://graph.microsoft.com/beta/security/informationProtection/sensitivityLabels"
$labels.value | Select-Object id, name, isActive |
    Export-Csv "$OutputPath\S3_Labels.csv" -NoTypeInformation

# ── S5 : Conditional Access — Politiques CA ──
Write-Host "[5/8] S5 — Politiques Conditional Access..." -ForegroundColor Cyan
$policies = Get-MgIdentityConditionalAccessPolicy
$copilotAppId = "fb8d773d-7ef8-4ec0-a117-179f88add510"
$copilotPolicies = $policies | Where-Object {
    $_.Conditions.Applications.IncludeApplications -contains $copilotAppId -or
    $_.Conditions.Applications.IncludeApplications -contains "All"
}
$copilotPolicies | Select-Object DisplayName, State,
    @{N='GrantControls';E={$_.GrantControls.BuiltInControls -join ","}} |
    Export-Csv "$OutputPath\S5_ConditionalAccess.csv" -NoTypeInformation

# ── S6 : Plugins — Applications consenties ──
Write-Host "[6/8] S6 — Applications OAuth consenties..." -ForegroundColor Cyan
$spApps = Get-MgServicePrincipal -All | Where-Object { $_.AppOwnerOrganizationId -ne $null }
$grants = Get-MgOauth2PermissionGrant -All
$grants | Select-Object ClientId, ConsentType, Scope, ResourceId |
    Export-Csv "$OutputPath\S6_OAuthGrants.csv" -NoTypeInformation

# ── S7 : Monitoring — Audit log Copilot ──
Write-Host "[7/8] S7 — Audit log CopilotInteraction..." -ForegroundColor Cyan
Connect-ExchangeOnline
$auditLogs = Search-UnifiedAuditLog -StartDate (Get-Date).AddDays(-7) `
    -EndDate (Get-Date) -Operations "CopilotInteraction" -ResultSize 100
$auditLogs | Select-Object CreationDate, UserIds, Operations |
    Export-Csv "$OutputPath\S7_AuditLog.csv" -NoTypeInformation

# ── S10 : Compliance — Rétention ──
Write-Host "[8/8] S10 — Politiques de rétention..." -ForegroundColor Cyan
Connect-IPPSSession
Get-RetentionCompliancePolicy | Select-Object Name, Enabled, Mode |
    Export-Csv "$OutputPath\S10_Retention.csv" -NoTypeInformation

# ── Résumé ──
Write-Host "`n═══ AUDIT TERMINÉ ═══" -ForegroundColor Green
Write-Host "Fichiers exportés dans : $OutputPath"
Get-ChildItem $OutputPath | Format-Table Name, Length -AutoSize
