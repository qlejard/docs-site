Mode d'emploi de ce guide
Comment utiliser ce document comme référentiel d'audit et de configuration de la gouvernance des identités
📋
Ce guide couvre la gouvernance des identités dans Microsoft Entra ID : revues d'accès, gestion des droits, activation privilégiée (PIM), workflows de cycle de vie. Il étend le guide Entra ID (qui couvre CA, MFA, Identity Protection) en se concentrant sur le cycle de vie et la gouvernance au-dessus des fondations.

Vérifiez chaque badge avant de recommander une feature.

Stratégie de déploiement recommandée

1
Quick Win — PIM & Access Reviews — ~30 min, +25 pts. Activer PIM sur les rôles Global Admin / Security Admin, créer une Access Review trimestrielle sur les groupes privilégiés.
2
Entitlement Management & ToU — ~1h, +15 pts. Créer un catalogue départemental, 2-3 Access Packages (onboarding Dev, onboarding RH), publier les Terms of Use via Conditional Access.
3
Lifecycle Workflows JML — ~2h, +12 pts. Configurer les workflows Joiner (attribution groupes + licence), Mover (changement département), Leaver (révocation accès + blocage).
4
Dashboard & Monitoring — ~1h, +8 pts. Activer Diagnostic Settings vers Log Analytics, déployer les KQL de surveillance (PIM activations, AR completion rate).
5
Admin Units, Verified ID & Guest Governance — Variable, +20 pts. Délégation RBAC scopée, intégration Verified ID dans les packages, lifecycle B2B avec sponsors.

🚨 Scénario d'attaque — Privilege creep
Un employé change de service mais conserve ses accès antérieurs. Après 3 ans, il cumule les droits de 4 services différents. Son compte compromis donne accès à 80% du SI.

💡 Distinction Access Reviews / PIM / Entitlement Management
Access Reviews (S1) = recertification — « cet accès est-il toujours justifié ? »
PIM (S3) = Just-In-Time — « active ce rôle pour 4h avec justification »
Entitlement Management (S2) = self-service — « demande un package d'accès via un catalogue »
💡 Pourquoi chaque paramètre compte

Chaque paramètre d'Access Review contrôle un maillon de la chaîne de recertification. Une fréquence trop longue laisse les accès dériver, un defaultDecision à « No change » maintient les accès non révisés, et l'absence de scope PIM exclut les rôles les plus critiques de la revue. En cas de compromission, ces paramètres déterminent le blast radius de l'attaque.

💡 Pourquoi c'est pertinent
  • Privilege creep — Sans recertification, les accès s'accumulent de manière monotone. Chaque mobilité interne ajoute sans jamais retirer.
  • Conformité — ISO 27001 A.9.2.5 exige une revue périodique des droits d'accès. SOC 2 CC6.1-CC6.3 idem.

⚠️ Piège fréquent — Auto-apply activé
Activer autoApplyDecisionsEnabled révoque automatiquement les accès non répondus. Si le reviewer est absent ou ne comprend pas la review, des accès légitimes sont supprimés. Gardez auto-apply désactivé et utilisez defaultDecision: Deny pour forcer la réponse.

1.2 — Configuration

Console : entra.microsoft.com → Identity Governance → Access Reviews

PowerShell — S1 — Config : Créer une Access Review sur les rôles PIM critiques
# ===================================================================
# S1 — Config : Créer une Access Review sur les rôles PIM critiques
# Prérequis : Connect-MgGraph -Scopes "AccessReview.ReadWrite.All"
# Module : Microsoft.Graph.Identity.Governance
# ===================================================================

$params = @{
    displayName = "Review trimestrielle - Rôles PIM T0/T1"
    descriptionForAdmins = "Recertification des éligibilités PIM Tier 0 et Tier 1"
    descriptionForReviewers = "Confirmez que cette personne a toujours besoin de ce rôle privilégié"
    scope = @{
        "@odata.type" = "#microsoft.graph.principalResourceMembershipsScope"
        principalScopes = @(@{
            "@odata.type" = "#microsoft.graph.accessReviewQueryScope"
            query = "/users"
            queryType = "MicrosoftGraph"
        })
        resourceScopes = @(@{
            "@odata.type" = "#microsoft.graph.accessReviewQueryScope"
            query = "/roleManagement/directory/roleDefinitions?$filter=displayName eq 'Global Administrator' or displayName eq 'Exchange Administrator' or displayName eq 'SharePoint Administrator' or displayName eq 'Security Administrator'"
            queryType = "MicrosoftGraph"
        })
    }
    reviewers = @(@{
        query = "/users/{SECURITY-ADMIN-ID}"
        queryType = "MicrosoftGraph"
    })
    settings = @{
        mailNotificationsEnabled = $true
        reminderNotificationsEnabled = $true
        justificationRequiredOnApproval = $true
        defaultDecisionEnabled = $true
        defaultDecision = "Deny"
        autoApplyDecisionsEnabled = $false
        recommendationsEnabled = $true
        recurrence = @{
            pattern = @{ type = "absoluteMonthly"; interval = 3 }
            range = @{ type = "noEnd"; startDate = "2026-04-01" }
        }
    }
}
New-MgIdentityGovernanceAccessReviewDefinition -BodyParameter $params

1.3 — Vérification

PowerShell — S1 — Vérification : Lister les Access Reviews et leur état
# ===================================================================
# S1 — Vérification : Lister les Access Reviews et leur état
# Prérequis : Connect-MgGraph -Scopes "AccessReview.Read.All"
# ===================================================================

Write-Host "📋 Access Reviews configurées" -ForegroundColor Cyan
$reviews = Get-MgIdentityGovernanceAccessReviewDefinition -All
$reviews | Format-Table DisplayName, Status, @{N='AutoApply';E={$_.Settings.AutoApplyDecisionsEnabled}}, @{N='DefaultDecision';E={$_.Settings.DefaultDecision}}, @{N='Recurrence';E={$_.Settings.Recurrence.Pattern.Type}}

# Vérifier les reviews sans réponse
$reviews | Where-Object { $_.Status -eq "InProgress" } | ForEach-Object {
    $instances = Get-MgIdentityGovernanceAccessReviewDefinitionInstance -AccessReviewScheduleDefinitionId $_.Id
    $instances | ForEach-Object {
        $decisions = Get-MgIdentityGovernanceAccessReviewDefinitionInstanceDecision -AccessReviewScheduleDefinitionId $_.Id -AccessReviewInstanceId $_.Id
        $notReviewed = ($decisions | Where-Object { $_.Decision -eq "NotReviewed" }).Count
        Write-Host "  ⚠️ Review '$($_.DisplayName)' : $notReviewed décisions en attente"
    }
}

💡 Pourquoi c'est pertinent
  • Self-service contrôlé — Les utilisateurs demandent eux-mêmes leurs accès via un portail (myaccess.microsoft.com) au lieu de tickets manuels. Réduit le délai de 5 jours à 1h.
  • Packages réutilisables — Un package “Développeur Frontend” regroupe 4 groupes + 2 apps + 1 site SharePoint. Attribution atomique, révocation atomique.

2.2 — Configuration

Console : entra.microsoft.com → Identity Governance → Entitlement Management → Catalogs

PowerShell — S2 — Config : Créer un catalogue + Access Package avec politique
# ===================================================================
# S2 — Config : Créer un catalogue + Access Package avec politique
# Prérequis : Connect-MgGraph -Scopes "EntitlementManagement.ReadWrite.All"
# Module : Microsoft.Graph.Identity.Governance
# ===================================================================

# 1. Créer un catalogue dédié
$catalog = New-MgEntitlementManagementCatalog -DisplayName "IT-Engineering" `
    -Description "Ressources pour les équipes Engineering" -IsExternallyVisible $false

# 2. Ajouter des ressources au catalogue (groupes, apps, sites)
New-MgEntitlementManagementCatalogResourceRequest -CatalogId $catalog.Id -BodyParameter @{
    requestType = "adminAdd"
    catalog = @{ id = $catalog.Id }
    resource = @{
        displayName = "SG-DevOps-Contributors"
        originId = "{GROUP-OBJECT-ID}"
        originSystem = "AadGroup"
    }
}

# 3. Créer un Access Package
$package = New-MgEntitlementManagementAccessPackage -DisplayName "Pack Développeur Frontend" `
    -Description "Accès Dev Frontend : Azure DevOps, GitHub, SharePoint Engineering" `
    -CatalogId $catalog.Id

# 4. Ajouter la politique d'attribution (approbation manager)
New-MgEntitlementManagementAssignmentPolicy -AccessPackageId $package.Id -BodyParameter @{
    displayName = "Politique standard - Approbation manager"
    allowedTargetScope = "specificDirectoryUsers"
    specificAllowedTargets = @(@{
        "@odata.type" = "#microsoft.graph.groupMembers"
        groupId = "{ALLOWED-GROUP-ID}"
    })
    expiration = @{ type = "afterDuration"; duration = "P365D" }
    requestorSettings = @{ enableTargetsToSelfAddAccess = $true; enableTargetsToSelfRemoveAccess = $true }
    requestApprovalSettings = @{
        isApprovalRequiredForAdd = $true
        stages = @(@{
            durationBeforeAutomaticDenial = "P7D"
            primaryApprovers = @(@{
                "@odata.type" = "#microsoft.graph.requestorManager"
                managerLevel = 1
            })
        })
    }
    reviewSettings = @{
        isEnabled = $true
        recurrenceType = "quarterly"
        startDateTime = "2026-04-01T00:00:00Z"
    }
}

# 5. Configurer Separation of Duties
# Empêcher cumul Pack Finance + Pack Audit
New-MgEntitlementManagementAccessPackageIncompatibleAccessPackageByRef `
    -AccessPackageId $package.Id `
    -OdataId "https://graph.microsoft.com/v1.0/identityGovernance/entitlementManagement/accessPackages/{INCOMPATIBLE-PKG-ID}"

2.3 — Vérification

PowerShell — S2 — Vérification : Inventaire des catalogues et packages
# ===================================================================
# S2 — Vérification : Inventaire des catalogues et packages
# Prérequis : Connect-MgGraph -Scopes "EntitlementManagement.Read.All"
# ===================================================================

Write-Host "📦 Catalogues Entitlement Management" -ForegroundColor Cyan
$catalogs = Get-MgEntitlementManagementCatalog -All
$catalogs | Format-Table DisplayName, State, IsExternallyVisible, CreatedDateTime

Write-Host "`n📋 Access Packages par catalogue" -ForegroundColor Cyan
foreach ($cat in $catalogs) {
    $packages = Get-MgEntitlementManagementAccessPackage -Filter "catalogId eq '$($cat.Id)'"
    Write-Host "  📁 $($cat.DisplayName) : $($packages.Count) package(s)"
    $packages | ForEach-Object {
        $policies = Get-MgEntitlementManagementAssignmentPolicy -Filter "accessPackageId eq '$($_.Id)'"
        $needsApproval = ($policies | Where-Object { $_.RequestApprovalSettings.IsApprovalRequiredForAdd }).Count
        $hasReview = ($policies | Where-Object { $_.ReviewSettings.IsEnabled }).Count
        Write-Host "    └─ $($_.DisplayName) | Approbation: $needsApproval/$($policies.Count) | Review: $hasReview/$($policies.Count)"
    }
}

Write-Host "`n🚫 Incompatibilités SoD configurées" -ForegroundColor Cyan
foreach ($pkg in (Get-MgEntitlementManagementAccessPackage -All)) {
    $incomp = Get-MgEntitlementManagementAccessPackageIncompatibleAccessPackage -AccessPackageId $pkg.Id -ErrorAction SilentlyContinue
    if ($incomp) { Write-Host "  $($pkg.DisplayName) ⛔ $($incomp.DisplayName -join ', ')" }
}

🚨 Scénario d'attaque — Standing privilege abuse
Un Global Admin utilise son compte privilégié permanent pour lire ses emails. Un phishing ciblé compromet la session. L'attaquant hérite immédiatement du rôle Global Admin — sans aucune barrière. Il désactive le MFA, crée un compte backdoor et exfiltre les données.

La durée d'activation, le MFA à l'élévation et l'approbation 4 yeux créent trois barrières successives qui réduisent le blast radius d'une attaque de > 90 %.

💡 Pourquoi c'est pertinent
  • Zero Standing Privilege — Un admin n'a aucun droit tant qu'il n'active pas son rôle. Le compte compromis d'un admin éligible ne donne rien à l'attaquant (MFA + approbation bloquent).
  • Blast radius temporel — Même si l'activation est compromise, le rôle expire en 1-4h. L'attaquant a une fenêtre limitée.
  • Audit complet — Chaque activation est loguée avec timestamp, justification, approbateur. Compatible avec les exigences SOC 2, ISO 27001 A.9.2.3.
⚠️ Piège fréquent — Break Glass non exempt
Les comptes Break Glass (urgence) doivent être les seuls avec un assignment permanent Global Admin. Si vous les mettez en éligible avec MFA + approbation, une panne MFA ou l'indisponibilité des approbateurs vous verrouille hors du tenant. Break Glass = permanent + exclus de PIM + exclus de Conditional Access + monitorés par alerte Sign-In.

# Helper : Configurer les settings PIM pour un rôle
function Set-PIMRoleSettings {
    param(
        [string]$RoleDisplayName,
        [string]$MaxActivationDuration,   # ISO 8601 : PT1H, PT4H, PT8H
        [bool]$RequireApproval,
        [bool]$RequireMFA,
        [bool]$RequireJustification,
        [string[]]$ApproverIds = @()
    )

    $roleDef = Get-MgRoleManagementDirectoryRoleDefinition -Filter "displayName eq '$RoleDisplayName'"
    $policyId = "DirectoryRole_$($roleDef.Id)_$($roleDef.Id)"

    # Règle d'activation
    $rules = @(
        @{
            "@odata.type" = "#microsoft.graph.unifiedRoleManagementPolicyExpirationRule"
            id = "Expiration_EndUser_Assignment"
            isExpirationRequired = $true
            maximumDuration = $MaxActivationDuration
            target = @{ caller = "EndUser"; operations = @("all"); level = "Assignment" }
        },
        @{
            "@odata.type" = "#microsoft.graph.unifiedRoleManagementPolicyAuthenticationContextRule"
            id = "AuthenticationContext_EndUser_Assignment"
            isEnabled = $RequireMFA
            claimValue = "c1"
            target = @{ caller = "EndUser"; operations = @("all"); level = "Assignment" }
        },
        @{
            "@odata.type" = "#microsoft.graph.unifiedRoleManagementPolicyEnablementRule"
            id = "Enablement_EndUser_Assignment"
            enabledRules = @($(if ($RequireJustification) { "Justification" }); $(if ($RequireMFA) { "MultiFactorAuthentication" })) | Where-Object { $_ }
            target = @{ caller = "EndUser"; operations = @("all"); level = "Assignment" }
        }
    )

    if ($RequireApproval -and $ApproverIds.Count -gt 0) {
        $rules += @{
            "@odata.type" = "#microsoft.graph.unifiedRoleManagementPolicyApprovalRule"
            id = "Approval_EndUser_Assignment"
            setting = @{
                isApprovalRequired = $true
                approvalStages = @(@{
                    approvalStageTimeOutInDays = 1
                    primaryApprovers = $ApproverIds | ForEach-Object {
                        @{ "@odata.type" = "#microsoft.graph.singleUser"; userId = $_ }
                    }
                })
            }
            target = @{ caller = "EndUser"; operations = @("all"); level = "Assignment" }
        }
    }

    foreach ($rule in $rules) {
        Update-MgPolicyRoleManagementPolicyRule -UnifiedRoleManagementPolicyId $policyId `
            -UnifiedRoleManagementPolicyRuleId $rule.id -BodyParameter $rule
    }

    Write-Host "✅ PIM configuré pour '$RoleDisplayName' : $MaxActivationDuration, Approval=$RequireApproval, MFA=$RequireMFA"
}

# --- Tier 0 : 1h + Approbation + MFA + Justification ---
$t0Roles = @("Global Administrator", "Exchange Administrator", "Security Administrator", "Privileged Role Administrator")
$approverIds = @("{SECURITY-ADMIN-1-ID}", "{SECURITY-ADMIN-2-ID}")
foreach ($role in $t0Roles) {
    Set-PIMRoleSettings -RoleDisplayName $role -MaxActivationDuration "PT1H" `
        -RequireApproval $true -RequireMFA $true -RequireJustification $true -ApproverIds $approverIds
}

# --- Tier 1 : 4h + MFA + Justification ---
$t1Roles = @("User Administrator", "Intune Administrator", "Teams Administrator", "SharePoint Administrator")
foreach ($role in $t1Roles) {
    Set-PIMRoleSettings -RoleDisplayName $role -MaxActivationDuration "PT4H" `
        -RequireApproval $false -RequireMFA $true -RequireJustification $true
}

# --- Tier 2 : 8h + Justification ---
$t2Roles = @("Helpdesk Administrator", "Reports Reader", "Message Center Reader")
foreach ($role in $t2Roles) {
    Set-PIMRoleSettings -RoleDisplayName $role -MaxActivationDuration "PT8H" `
        -RequireApproval $false -RequireMFA $false -RequireJustification $true
}

# [4/10] S4 — Lifecycle Workflows
# =============================================
Write-Host "`n🔄 [4/10] Export Lifecycle Workflows..." -ForegroundColor Yellow
$workflows = Get-MgIdentityGovernanceLifecycleWorkflow -All
$wfData = $workflows | Select-Object Id, DisplayName, Category, IsEnabled, IsSchedulingEnabled,
    @{N='LastModified';E={$_.LastModifiedDateTime}},
    @{N='TaskCount';E={$_.Tasks.Count}}
$wfData | Export-Csv "$ExportPath\S4_LifecycleWorkflows.csv" -NoTypeInformation -Encoding UTF8
Write-Host "  ✅ $($workflows.Count) workflows"

# Recent executions
$wfRuns = Get-MgIdentityGovernanceLifecycleWorkflowRun -All -Top 50
$wfRuns | Select-Object Id, WorkflowId,
    @{N='Status';E={$_.ProcessingStatus}},
    @{N='StartedDateTime';E={$_.StartedDateTime}},
    @{N='TotalUsers';E={$_.TotalUsersCount}},
    @{N='FailedUsers';E={$_.FailedUsersCount}} |
    Export-Csv "$ExportPath\S4_WorkflowRuns.csv" -NoTypeInformation -Encoding UTF8
Write-Host "  ✅ $($wfRuns.Count) exécutions récentes"

# =============================================
# [5/10] S5 — Governance Dashboard KPIs
# =============================================
Write-Host "`n📊 [5/10] KPIs Governance Dashboard..." -ForegroundColor Yellow
$kpis = [ordered]@{
    "Access Reviews actives"     = ($reviews | Where-Object Status -eq "InProgress").Count
    "Access Packages publiés"    = ($packages | Where-Object { -not $_.IsHidden }).Count
    "Éligibilités PIM"           = $eligible.Count
    "Assignments permanentes"    = $permanent.Count
    "Lifecycle Workflows actifs" = ($workflows | Where-Object IsEnabled).Count
}
$kpis.GetEnumerator() | Select-Object @{N='KPI';E={$_.Key}}, @{N='Valeur';E={$_.Value}} |
    Export-Csv "$ExportPath\S5_KPIs.csv" -NoTypeInformation -Encoding UTF8
$kpis.GetEnumerator() | ForEach-Object { Write-Host "  📈 $($_.Key) : $($_.Value)" }

# =============================================
# [6/10] S6 — Terms of Use
# =============================================
Write-Host "`n📜 [6/10] Export Terms of Use..." -ForegroundColor Yellow
$tous = Get-MgIdentityGovernanceTermsOfUseAgreement -All
$tous | Select-Object Id, DisplayName, IsPerDeviceAcceptanceRequired, IsViewingBeforeAcceptanceRequired,
    @{N='UserReacceptFrequency';E={$_.TermsExpiration.Frequency}} |
    Export-Csv "$ExportPath\S6_TermsOfUse.csv" -NoTypeInformation -Encoding UTF8
Write-Host "  ✅ $($tous.Count) Terms of Use"

# =============================================
# [7/10] S7 — Administrative Units
# =============================================
Write-Host "`n🏢 [7/10] Export Administrative Units..." -ForegroundColor Yellow
$aus = Get-MgDirectoryAdministrativeUnit -All
$auData = $aus | Select-Object Id, DisplayName,
    @{N='MembershipType';E={$_.MembershipType}},
    @{N='MembershipRule';E={$_.MembershipRule}},
    @{N='IsMemberManagementRestricted';E={$_.IsMemberManagementRestricted}}
$auData | Export-Csv "$ExportPath\S7_AdministrativeUnits.csv" -NoTypeInformation -Encoding UTF8
Write-Host "  ✅ $($aus.Count) Administrative Units"

# =============================================
# [8/10] S8 — Verified ID
# =============================================
Write-Host "`n🆔 [8/10] Vérification Verified ID..." -ForegroundColor Yellow
$vcAudits = Get-MgAuditLogDirectoryAudit -All `
    -Filter "loggedByService eq 'Verified ID' and activityDateTime ge $since30d" `
    -Top 20 -ErrorAction SilentlyContinue
if ($vcAudits) {
    $vcAudits | Select-Object ActivityDateTime, ActivityDisplayName, Result |
        Export-Csv "$ExportPath\S8_VerifiedID_Audit.csv" -NoTypeInformation -Encoding UTF8
    Write-Host "  ✅ $($vcAudits.Count) événements Verified ID (30j)"
} else {
    Write-Host "  ℹ️ Aucun événement Verified ID détecté"
}

# =============================================
# [9/10] S9 — Guest Governance
# =============================================
Write-Host "`n👥 [9/10] Export Guest Governance..." -ForegroundColor Yellow
$guests = Get-MgUser -All -Filter "userType eq 'Guest'" -Property Id, DisplayName, Mail, UserPrincipalName, CreatedDateTime, SignInActivity, ExternalUserState
$guestData = $guests | Select-Object Id, DisplayName, Mail, ExternalUserState,
    @{N='CreatedDateTime';E={$_.CreatedDateTime}},
    @{N='LastSignIn';E={$_.SignInActivity.LastSignInDateTime}},
    @{N='DaysSinceLastSignIn';E={
        if ($_.SignInActivity.LastSignInDateTime) {
            [math]::Round(((Get-Date) - $_.SignInActivity.LastSignInDateTime).TotalDays)
        } else { "Jamais" }
    }}
$guestData | Export-Csv "$ExportPath\S9_Guests.csv" -NoTypeInformation -Encoding UTF8
$inactive = ($guestData | Where-Object { $_.DaysSinceLastSignIn -ne "Jamais" -and $_.DaysSinceLastSignIn -gt 90 }).Count
Write-Host "  ✅ $($guests.Count) guests ($inactive inactifs >90j)"

# Invitation policy
$policy = Get-MgPolicyAuthorizationPolicy
$guestInvite = $policy.AllowInvitesFrom
Write-Host "  📋 Politique invitation : $guestInvite"

Write-Host "`n📊 [10/10] Résumé compliance..." -ForegroundColor Yellow
$summary = [ordered]@{
    "Date d'audit"                = Get-Date -Format "yyyy-MM-dd HH:mm"
    "Tenant ID"                   = (Get-MgContext).TenantId
    "Access Reviews"              = $reviews.Count
    "Catalogues Entitlement"      = $catalogs.Count
    "Access Packages"             = $packages.Count
    "Assignments actives"         = $assignments.Count
    "PIM Éligibilités"            = $eligible.Count
    "PIM Permanentes"             = $permanent.Count
    "PIM Activations (30j)"       = $activations.Count
    "Lifecycle Workflows"         = $workflows.Count
    "Terms of Use"                = $tous.Count
    "Administrative Units"        = $aus.Count
    "Guests total"                = $guests.Count
    "Guests inactifs >90j"        = $inactive
    "Politique invitation guests" = $guestInvite
}
$summary.GetEnumerator() | Select-Object @{N='Métrique';E={$_.Key}}, @{N='Valeur';E={$_.Value}} |
    Export-Csv "$ExportPath\S10_Summary.csv" -NoTypeInformation -Encoding UTF8
