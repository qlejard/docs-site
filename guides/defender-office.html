Mode d'emploi de ce guide
Perimetre, hors-perimetre et methode de deploiement incremental
ℹ️
Ce guide couvre strictement MDO : authentification email (SPF, DKIM, DMARC), EOP, Safe Attachments, Safe Links, anti-phishing, Attack Simulation Training, investigation, quarantaine, integration Defender XDR et reporting. Hors perimetre : administration Exchange generaliste, Purview DLP, securite Teams hors flux email, gouvernance Entra ID.

Strategie de deploiement recommandee

1
Quick Win (<=15 min) : presets Standard + activation Safe Links/Safe Attachments + quarantaine admin.
2
Durcissement MDO : impersonation, mailbox intelligence, protection URL avancee, TABL.
3
Maturite P2 : Attack Simulation Training, Threat Explorer, AIR et correlation XDR.
4
Pilotage continu : Configuration Analyzer, rapports CISO, routines hebdo/mensuelles.
⚠️
Solutions ecartees : produit tiers SEG comme controle principal. Raisons : duplication de policy, perte de telemetrie native MDO/XDR, hausse de cout et complexite d'escalade.

1
Email Authentication (SPF, DKIM, DMARC, ARC, BIMI) EOP MDO P1+
S1 - Criticite 🔴 : prouver l'identite de l'expediteur (cachet et signature officielle)
🚨
Sans SPF/DKIM/DMARC, un attaquant peut usurper votre domaine et envoyer un faux ordre de virement. Analogie : cachet et signature officielle absents, la lettre parait legitime mais personne ne peut verifier l'emetteur.
ParametreDefaut⚡ Quick Win🔒 Avance🎯 OptimalLicence
SPF TXTAbsent ou ~all permissifv=spf1 include:spf.protection.outlook.com ~allInventaire expditeurs tiers-all apres stabilisationEOP
DKIM SigningDesactiveEnable DKIMRotation semestrielleRotation + monitoring failEOP
DMARC PolicyAbsentep=nonep=quarantine; pct=50p=reject; pct=100EOP
DMARC RUA/RUFAucun reportingRUA activeRUA+RUF activeFeed SIEM + revue mensuelleExterne
ARC / BIMINon exploiteEvaluation ARCARC trust chainBIMI + logo valideDNS/Branding
💡
Pourquoi chaque parametre : SPF reduit spoofing basique, DKIM signe cryptographiquement les messages, DMARC impose la politique d'action et fournit la visibilite RUA/RUF, ARC preserve la confiance en forward, BIMI ajoute un signal de marque mais ne remplace pas DMARC.
⚠️
Piege courant : passer directement a p=reject sans phase p=none puis p=quarantine casse des flux legitimes (CRM, ticketing, MFP).

Configuration step-by-step (PowerShell + DNS)

1
Portail : security.microsoft.com > Email & collaboration > Policies & rules > Threat policies.
2
PowerShell : connexion Exchange Online puis activation DKIM sur chaque domaine accepte.
3
DNS : publier SPF puis DMARC (p=none), monitorer 2-4 semaines, migrer vers quarantine puis reject.
PowerShell - SPF/DKIM/DMARC (Exchange Online)
# prerequis
Install-Module ExchangeOnlineManagement -Scope CurrentUser
Connect-ExchangeOnline

# 1) DKIM - creer puis activer (remplacer contoso.com)
New-DkimSigningConfig -DomainName contoso.com -Enabled $false
Set-DkimSigningConfig -Identity contoso.com -Enabled $true

# 2) Recuperer les CNAME DKIM a publier
Get-DkimSigningConfig -Identity contoso.com | Format-List Domain,Selector1CNAME,Selector2CNAME,Enabled

# 3) SPF (DNS TXT)
# Host: @
# Value: v=spf1 include:spf.protection.outlook.com ~all

# 4) DMARC (DNS TXT)
# Host: _dmarc
# Value phase 1: v=DMARC1; p=none; rua=mailto:dmarc-rua@contoso.com; ruf=mailto:dmarc-ruf@contoso.com; pct=100; adkim=s; aspf=s
# Value phase 2: v=DMARC1; p=quarantine; rua=mailto:dmarc-rua@contoso.com; pct=100
# Value phase 3: v=DMARC1; p=reject; rua=mailto:dmarc-rua@contoso.com; pct=100

Verification

Verification DKIM + suivi DMARC
Get-DkimSigningConfig | Select-Object Domain,Enabled,Status
Resolve-DnsName -Type TXT contoso.com
Resolve-DnsName -Type TXT _dmarc.contoso.com

# Controle attendu
# SPF present avec include:spf.protection.outlook.com
# DKIM Enabled=True
# DMARC present avec p=none/quarantine/reject selon phase

Safe Links MDO Plan 1+
S4 - Criticite 🟠 : verification de l'adresse a la volee
🚨
Le lien peut devenir malveillant apres reception. Safe Links protege au clic (time-of-click), pas uniquement a la livraison.
ParametreDefaut⚡🔒🎯Licence
EnableSafeLinksForEmailOff/partielOnOnOn tous usersP1+
DoNotRewriteUrlsExceptions largesExceptions limiteesExceptions minimiseesExceptions justifiees uniquementP1+

ScanUrlsInOfficeAppsOffOnOnOn + hardening macrosP1+
Teams URL ProtectionPartielOnOnOn + TABL URL feedP1+
💡
La valeur de Safe Links est temporelle : detection au clic. Les exceptions de reecriture sont le principal angle mort.
⚠️
Une whitelist URL permanente dans TABL peut neutraliser la protection Safe Links. Ajouter une date d'expiration systematique.

Configuration

Connect-ExchangeOnline

New-SafeLinksRule -Name "MDO-SL-Strict-Rule" -SafeLinksPolicy "MDO-SL-Strict" -RecipientDomainIs contoso.com

Script de campagne AST (Graph API)

PowerShell - AST campaign via Microsoft Graph
# Prerequis: App registration avec permissions AttackSimulation.ReadWrite.All
# et module Microsoft.Graph

Connect-MgGraph -Scopes "AttackSimulation.ReadWrite.All,User.Read.All"
# Microsoft.Graph 2.x : appel direct du endpoint /beta via Invoke-MgGraphRequest
# (Select-MgProfile a ete supprime du module a partir de la v2)

$payload = @{
  displayName = "AST-Phishing-Mensuel-Contoso"
  description = "Simulation credential harvest + training"
  payloadDeliveryPlatform = "email"
  attackType = "credentialHarvest"
  includedAccountTarget = @{
    type = "addressBook"
    accountTargetEmails = @("finance@contoso.com","achats@contoso.com")
  }
  endUserNotificationSetting = @{
    settingType = "training"
    positiveReinforcement = $true
  }
  launchDateTime = (Get-Date).AddDays(7).ToString("o")
} | ConvertTo-Json -Depth 6

Invoke-MgGraphRequest -Method POST -Uri "https://graph.microsoft.com/beta/security/attackSimulation/simulations" -Body $payload -ContentType "application/json"
