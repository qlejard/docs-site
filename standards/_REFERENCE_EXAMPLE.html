1.1 — Tableau de configuration complet

ParamètreDéfaut⚡ Recommandé🎯 Best PracticeLicence
ProtectionActivee False True True Inclus
NiveauDetection 1 3 4 Standard
ActionSurDetection NoAction Quarantine Quarantine Inclus
JournalisationActivee True True True Inclus
NotificationAdmin False False True + email admin Premium
💡
Pourquoi chaque paramètre compte :
  • ProtectionActivee : sans activation, aucune détection n'est effectuée — les menaces passent intégralement.
  • NiveauDetection (3-4) : le niveau 1 ne détecte que les menaces évidentes. Le niveau 3+ active les heuristiques ML qui couvrent les menaces 0-day. Le niveau 4 est plus agressif (plus de faux positifs mais moins de fuites).
  • ActionSurDetection = Quarantine : sans action, la détection est inutile — l'élément malveillant reste accessible à l'utilisateur.
  • NotificationAdmin : les presets ne l'activent jamais. Sans notification, l'admin ne sait pas qu'une menace a été détectée → pas d'investigation, pas de remédiation proactive.
⚠️
Piège classique : Le paramètre NotificationAdmin n'est jamais activé par les presets. C'est le seul paramètre qui nécessite une configuration manuelle via custom policy. Sans lui, les détections passent inaperçues de l'équipe IT.

1.2 — Configuration

PowerShell — Configuration de la protection
# Configurer la protection avec les best practices
Set-ProtectionPolicy -Identity "Policy-Custom" `
    -ProtectionActivee $true `
    -NiveauDetection 3 `
    -ActionSurDetection Quarantine `
    -NotificationAdmin $true `
    -AdminEmail "admin@$domain"

# Alternative : Az CLI
az protection policy update --name "Policy-Custom" `
    --protection-enabled true `
    --detection-level 3 `
    --action quarantine

1.3 — Vérification

PowerShell — Vérification
Get-ProtectionPolicy | Format-List Name, ProtectionActivee, `
    NiveauDetection, ActionSurDetection, NotificationAdmin

# Résultat attendu :
# ProtectionActivee   : True
# NiveauDetection     : 3
# ActionSurDetection  : Quarantine
# NotificationAdmin   : True
